# ✅ 一、平台角色设计
### 1. 主办方（Organizer）
* 发布 Hackathon
* 审核赞助商
* 管控活动流程
* 审核参赛队伍
* 开启/结束阶段（报名/签到/投票）
* 奖金池管理与发放
### 2. 赞助商（Sponsor）
* 创建赞助信息（奖金、代币、NFT、礼品）
* 向奖金池存入资金
* 获取赞助权益（展示、Logo、投票权等）
* 实时查看活动数据
### 3. 参与者（Participant）
* 创建队伍/加入队伍
* 报名活动
* 现场签到（链上或链下签名）
* 提交作品（代码链接、Demo、文档）
* 参与投票（如果主办方允许）

# ✅ 二、系统模块拆解（核心模块）
下面是你平台应该包含的完整模块功能。
---
# **📦 1. 活动管理模块 (Event Management)**
### 主办方功能
* 创建新 Hackathon（链上/链下）
  * 活动名称、地点、时间
  * 比赛阶段（报名→签到→提交→投票→颁奖）
  * 奖项配置（一等奖/二等奖/三等奖等）
  * 是否允许赞助商投票、是否允许公众投票
---
# **📦 2. 赞助商/奖金池模块（Sponsor & Funding Pool）**
### 赞助商功能
* 发起赞助（支持多种形式）
  * ERC20 代币
  * 原生资产（ETH/SOL 等）
  * NFT（奖品）
* 储存资金至奖金池合约
### 奖金池功能
* 多币种托管（ERC20 / SOL / 原生资产）
* 奖励比例配置
* 锁仓直到活动结束
* 自动发奖（智能合约）
---
# **📦 3. 报名与队伍管理（Register & Team Management）**
### 参与者功能
* 连接钱包登录
* 创建队伍 / 加入队伍
* 填写队伍资料（成员名单、技能）
* 报名活动
* 签署报名 SBT（不能转让）
### 主办方功能
* 审核/拒绝队伍
* 管理队伍人数
* 查看队伍资料
---
# **📦 4. 现场签到模块（Check-in System）**
### 链下签名验证（推荐）
* 用户扫二维码 → 发起签名 → 后端验证 → 链上记录签到
* 成本低，安全性高
### 主办方功能
* 查看签到人数
* 导出签到记录
* 防止重复签到
---
# **📦 5. 作品提交模块（Submission System）**
### 参与者功能
* 提交作品信息
  * GitHub Repo
  * Demo 视频
  * 项目文档
* 上传附件（链下存储，例如 IPFS/Arweave）
### 链上
* 作品指纹（Hash）
* 时间戳证明（不可篡改）
### 主办方功能
* 管理作品
* 标记项目为通过/不通过
---
# **📦 6. 投票系统（Voting System）**
你可以做三个维度的投票：
### ① 评委投票（常规）
* 白名单地址
* 一人一票或权重票
### ② 赞助商投票
* 根据赞助金额赋予权重
* 例如：1 USDC = 1 票（可配置）
### ③ 公众投票（可选）
* 限制每地址投票次数
* 防刷票机制
* 投票需要持有活动 NFT 才能投票（防假用户）
### 投票方式
* 链上投票（透明）
* 链下签名 → 统一上链（节省 Gas）
---
# **📦 7. 排名与结果模块（Judging & Results）**
* 根据投票自动计算排名
* 公布结果（链上记录）
* 评委评价系统（文本、评分）
* 链上证明（Result SBT）
---
# **📦 8. 奖金发放模块（Payout System）**
**核心：自动化发奖（最重要）**
奖金池合约支持：
* 多币种发放
* 自动按比例分配给队长或全队
* 主办方触发发奖（避免自动触发安全风险）
* 添加防止重复发放机制
---
# 🎨 9. 前端模块（UI/UX）
* 首页活动展示
* 活动详情页（时间线 + 奖金池展示）
* 报名页面
* 投票页面
* 结果展示页面（排行榜）
* 用户中心（参赛记录、证书）
* 赞助商后台
* 主办方后台
* 所有后台统一一个网站，区分不同的角色
---
# 🛠️ 技术栈
## 前端
- **框架**: React
- **Web3 交互**: Web3.js 或 Ethers.js
## 后端
- **语言**: Go（可选，部分功能可直接通过智能合约实现）
## 智能合约
- **语言**: Solidity
- **部署链**: Ethereum、Polygon 等
## 存储
- **分布式存储**: IPFS（用于存储项目代码和演示文件）
## 身份验证
- **钱包集成**: MetaMask 或其他 Web3 钱包
